# PZ Deep Research 美股研究产品需求文档

## 1. 文档状态

- 状态：阶段 C 数据骨架已实现，候选筛选与 C 端入口尚未实现
- 首发领域：美股二级市场研究
- 目标用户：需要公开信息、证据链和结构化分析的 C 端用户
- 默认输出语言：简体中文，保留英文公司名、Ticker 和原始来源

当前工程进度：已建立版本化金融 Schema、Ticker/Exchange/CIK 精确解析与内存缓存、SEC 证券目录/Submissions/Company Facts 适配器、SerpApi Google Finance/News 适配器和 fixture 驱动的单公司领域级闭环。金融 Runtime 尚未注册到公开 API，不会被 C 端误用为已完成产品。

## 2. 产品定位

PZ 美股研究不是交易信号或自动荐股产品。它将用户从公司、行业、主题、事件、宏观变量或财务条件切入的问题，转换为可追溯的公开信息研究，并在证据充分时收敛为 1–3 支值得继续研究的美股候选；证据不足时允许输出 0 支。

固定产品边界：

- 输出“研究候选”，不输出个性化买入、卖出、仓位或止损指令。
- 不承诺回报，不把目标价或预期收益率作为 MVP 输出。
- 所有时效性结论都必须标注 `as_of`，所有关键事实和数字都必须可回溯到来源。
- 最终报告必须明确声明“仅供信息研究与学习参考，不构成投资建议”。

## 3. MVP 范围

### 3.1 支持

- 市场：NYSE、Nasdaq、NYSE American 的美国普通股。
- 研究周期：默认 6–18 个月的中长期基本面研究。
- 输入类型：明确公司、行业/主题、宏观/事件、财务条件。
- 候选漏斗：生成 5–10 支初始候选，深度研究 3–5 支，最终输出 0–3 支。
- 证据：SEC EDGAR、公司 Investor Relations、SerpApi Google Finance、SerpApi Google News 及可访问的新闻/网页正文。

### 3.2 不支持

- ETF、OTC、期权、期货、债券、加密资产和自动交易。
- 分钟级或日内交易研究、盘口信号和技术分析。
- 个性化仓位、交易执行、税务或完整投资组合建议。
- 完整三张表预测、分析师一致预期、量化回测和组合优化。
- 声称已扫描全部美股。MVP 必须向用户展示实际支持的股票池边界。

## 4. 用户流程

1. 用户输入自然语言问题。
2. 系统生成并展示结构化的问题解释：输入类型、市场、研究时点、时间范围和关键维度。
3. 只有当公司或市场范围存在致命歧义时才请求一次追问；其余情况使用明示默认值继续。
4. 系统构建候选股票池，展示候选来源和初步入选理由。
5. 系统并行执行基本面、行业/事件和风险研究。
6. 确定性计算引擎生成财务指标、增长、估值和横向比较。
7. 筛选引擎按硬门槛淘汰不可用标的，并使用版本化方法对其余候选排名。
8. 报告 Agent 只使用结构化证据和计算结果写作；Verifier 检查数字、引用、时效、反方证据和输出边界。
9. 用户首先看到 0–3 支候选卡片，再按需阅读完整报告和来源。

## 5. 最终输出

每支候选股必须包含：

- 公司名称、Ticker、交易所。
- 与用户问题的关联和入选理由。
- 关键财务指标、数据期间和来源。
- 估值语境，包括同行或自身历史比较，不得只给孤立倍数。
- 潜在催化剂、主要风险、反方观点和逻辑失效条件。
- 研究时点、数据新鲜度、证据完整度和研究置信度。
- 来源列表、方法局限和固定免责声明。

## 6. 数据与证据规则

来源优先级：

```text
SEC / 公司正式披露
> 交易所与美国政府机构
> 结构化市场数据
> 主流新闻与专业媒体
> 聚合平台
> 其他公开网页
```

规则：

- SEC XBRL 或正式 filing 是已披露财务数字的优先事实源。
- Google Finance 和其他聚合数据用于行情、发现、辅助展示和交叉检查，不得无声覆盖一手披露。
- 重要数据冲突必须保留冲突、记录选择规则，并在会影响结论时向用户披露。
- 价格、新闻和时效事件必须记录原始时区和 UTC 时间。
- 任何数值变换都必须保留原始值、单位、币种、公式和方法版本。

## 7. Multi-Agent 边界

MVP 只设三个受限角色：

1. `Planner`：输出结构化问题解释、候选构建计划和研究维度。
2. `Financial Analyst`：使用指定工具和证据仓生成候选公司研究档案。
3. `Verifier`：只输出结构化检查结果，不直接修改或替换报告。

Agent 不得无限对话，不得相互引用未入库结论。工作流负责并发、超时、预算、重试、检查点和最大重写次数。

## 8. MVP 成功标准

- 明确公司输入能稳定解析到唯一 Ticker、Exchange 和 CIK。
- 任何展示的财务数字都可追溯到原始披露和报告期。
- 任何候选都具有正方证据、反方证据、风险和逻辑失效条件。
- 证据不足的评测样例能稳定输出 0 支，而不是强制荐股。
- 相同输入、相同数据快照和相同方法版本下，硬门槛与计算结果可重现。
- 报告不包含无来源数字、无依据目标价、回报承诺或个性化仓位指令。

## 9. 上线前门槛

- 完成目标市场的数据许可和投资研究/投资建议合规评估。
- 完成一组覆盖公司、主题、宏观、新闻和证据不足的固定评测集。
- 完成数字正确性、引用支持性、时效性、Ticker 解析、0 候选降级和报告边界验收。
- 向用户清晰展示股票池边界、数据截止时间、方法局限和免责声明。
